# Conventions

Pins the rules that keep the doc tree and the codebase mutually legible. New pages and new code MUST follow these — drift is reviewed in PR.

## Doc tree

### Diátaxis quadrants

Every page under `docs/` belongs to exactly one of:

| Quadrant | Frontmatter `type` | Purpose |
| --- | --- | --- |
| Tutorial | `tutorial` | Learning by doing, ordered steps, no skipping |
| How-to | `how-to` | Recipe for a specific task, may be skimmed |
| Reference | `reference` | Look-up, exhaustive, neutral |
| Explanation | `explanation` | Why it is shaped this way, not how to use it |

Mixing two quadrants in one page is a smell. Split.

### Frontmatter

Every page MUST start with:

```yaml
---
type: tutorial | how-to | reference | explanation
updated: YYYY-MM-DD
---
```

Optional keys: `status: snapshot | draft` (for time-bounded pages like audits).

### Naming

- Pages under `docs/`: `kebab-case.md`.
- How-to slugs are **verb-first**: `manage-workspaces`, `configure-claude-settings`, `grant-out-of-scope-path`. Not `workspaces-management`.
- No spaces. No uppercase outside frontmatter values.

### Folder structure

```
docs/
  index.md              ← Diátaxis MOC, canonical entry point
  CONVENTIONS.md        ← this file
  HEALTH.md             ← auto-generated freshness telemetry
  concepts.md           ← canonical vocabulary
  getting-started.md    ← single tutorial
  keyboard-shortcuts.md ← reference
  how-to/               ← all how-to pages live here, never at docs/ root
    *.md
  architecture/         ← all explanation pages about internals
    *.md
```

How-to pages MUST live under `docs/how-to/`, never at `docs/` root. Architecture pages MUST live under `docs/architecture/`.

### Link discipline

- Relative links only. Never absolute (`https://github.com/...` for source files is forbidden — use the relative path).
- Page-to-page links pass through `docs/index.md` when possible — the MOC stays the canonical map.
- Every external reference (concept, library, RFC) is linked once with the source URL.

## Code

### TypeScript

- Strict mode enforced via `tsconfig.json`. Diagnostics in `pnpm typecheck` MUST be zero.
- `any` forbidden outside explicit boundaries (`unknown` + Zod parse, or a typed cast with a `// safe because…` line).
- Zod schemas in `packages/shared/src/ipc.ts` are the source of truth for IPC types. `z.infer` at consumption sites.

### File naming

- Modules: `kebab-case.ts`.
- React components: `PascalCase.tsx`.
- Hooks: `useCamelCase.ts`.
- Tests: colocated `*.test.ts` for unit (vitest). E2E in `e2e/*.spec.ts` (Playwright).

### Comments

Default: none. Names + types document what the code does. Comments only for non-obvious WHY: a hidden constraint, a workaround, a surprising invariant. If removing a comment would not confuse a future reader, do not write it.

## Commits — Conventional Commits (Angular)

Format: `<type>(<scope>): <subject>`.

| Type | When |
| --- | --- |
| `feat` | New user-facing feature |
| `fix` | Bug fix |
| `docs` | Documentation only |
| `chore` | Tooling, deps, build scripts, no production code change |
| `refactor` | Code restructure with no behavior change |
| `test` | Add or improve tests, no production code change |
| `perf` | Performance improvement |
| `build` | Build system or external dependencies |
| `ci` | CI configuration |
| `style` | Formatting only (no logic) |

Scope is optional but encouraged: `feat(renderer):`, `fix(main):`, `docs:`, `chore(deps):`.

Body is optional. When present, separate from subject by blank line. Wrap at 72 columns.

Breaking changes: append `!` after type/scope (`feat(ipc)!:`) and add a `BREAKING CHANGE:` footer.

### Examples

```
feat(renderer): markdown notes panel as 4th tab in Settings
fix(main): conpty_console_list_agent.js crash is cosmetic, do not surface
docs: pin Diátaxis quadrants in docs/CONVENTIONS.md
refactor(shared): rename SessionType enum members to PascalCase
chore(deps): bump @xterm/xterm to 5.5.0
```

### Husky enforcement

A pre-commit hook running `commitlint --edit` is **opt-in** — declared here, not auto-installed. To enable locally:

```bash
pnpm dlx husky init
echo "pnpm dlx commitlint --edit \$1" > .husky/commit-msg
```

The hook is not installed by default to avoid forcing the convention on contributors who only patch a typo. CI may enforce later (separate decision).

## Changelog

`CHANGELOG.md` follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) + Conventional Commits. Sections per release: `Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, `Security`, `Docs`. Pre-release entries land under `## [Unreleased]`.

## Doc-health telemetry

`docs/HEALTH.md` is generated by `pnpm docs:health` (`scripts/docs-health.ts`). It reports commit ratio `apps/*` vs `docs/*`, broken links, and per-page freshness (`updated:` vs last `git log` mtime on the file). A 60-day observation window precedes any formal maintenance policy — see HEALTH.md preamble.
